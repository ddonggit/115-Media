# AGENTS.MD 通用规则
## 1. 全局上下文加载与依赖定位
- 利用 OpenCode 的代码库扫描能力，将完整的项目结构、核心代码及配置全量投喂给 DeepSeek-V4-Pro。
- 在执行任务前，必须从百万字上下文中精准提取当前任务涉及的跨文件依赖关系与核心逻辑入口，禁止基于局部信息盲目修改。
## 2. 深度思考与工具调用严格分离
- 在执行任何代码修改、文件操作或终端命令前，必须先在思考过程中完整推演执行意图、潜在副作用及验证逻辑。
- 禁止在思维链中夹杂工具调用指令，必须等待思考完成后，再调用 OpenCode 的文件读写或终端执行工具。
## 3. 长程多步任务分阶与终端验证
- 将复杂任务拆解为“探索-规划-执行-验证”四个阶段。V4-Pro 负责复杂多步推演与架构规划，OpenCode 负责落地执行。
- 每一步代码变更后，必须通过 OpenCode 终端运行构建、Lint 或测试命令进行真实环境验证，禁止仅靠模型推理断定修改成功。
## 4. 最小化变更与防过度重构
- 严格限制单次变更的代码范围，仅针对当前任务目标进行修改。
- 禁止利用 V4-Pro 的长上下文全局理解能力进行未被要求的“顺手重构”或“代码风格统一”，防止引入隐蔽破坏。
## 5. 异常捕获与回滚机制
- 当终端命令执行报错或测试未通过时，必须先分析错误日志，调整思路后重新尝试。
- 若连续 2 次修复失败，必须停止操作，输出当前诊断结论，并提示通过 OpenCode 的 Git 回滚功能恢复至上一个稳定状态。
